#### 特殊之处
* ECMAScript/JavaScript的私有的字段，在TypeScript中不能使用 private, protected, public, abstract，因为是没有意义的
* TypeScript 在类上的所有修饰符，只是在 TypeScript 中做了一些额外的代码，使得这个语法成立，但事实上对于 JavaScript 来说，它是存在的
* 公共的 public 可加可不加，它是缺省的，Y+ 是指在初始化类实例时可以使用特殊的语法来声明类的公共参数

* 类的构造方法可以使用 private 修饰符，这样不能直接被 new 出来，但可以被类的静态方法间接创建出来，类似于工厂函数
* 类的构造方法可以使用 protected 修饰符，这样不能直接被 new 出来，但可以被类的静态方法间接或者子类创建出来，类似于工厂函数
  > 类的构造方法缺省的情况下时 public
* abstract 的构造方法不能够使用，是因为这样会指一个它的子类不能被构建，它的“抽象”应该是指子类自身是抽象的，而不是指自身是不是抽象的，这样会使它和它的子类都无法被 new 出来，所以 abstract 必须写在类声明上，这样才有意义
* 其他
  1. 类的参数 可选语法
  2. 类的参数 只读语法
  3. 类的方法中参数可以可选，但是必须是最后一个
